# Dragon Ball Android Application

# Overview
This project is a Dragon Ball Android application developed using a modern Android tech stack and structured following Clean Architecture principles in a multi-module setup.
The main goal of this project was to demonstrate strong architectural knowledge, separation of concerns, scalability, and maintainability of code.
The application consumes remote data, stores selected data locally, and presents it through a clean and responsive UI.

# Architecture
- The project is built using:
- Clean Architecture
- Clear separation between domain, data, and presentation layers
- Strict dependency rule (outer layers depend on inner layers)
- Multi-module structure
- Improved scalability
- Better code organization
- Clear module responsibilities

This structure was intentionally chosen to ensure long-term maintainability and production-level code quality.

# Tech Stack
- Koin – Dependency Injection
- Retrofit – HTTP networking and API communication
- Room – Local database persistence
- Coil – Image loading
- Kotlin + Coroutines – Asynchronous programming
- Jetpack components – ViewModel, etc.
